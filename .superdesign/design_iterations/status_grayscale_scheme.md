# 状态仅用灰度层级区分方案（微调说明）

仪表盘与表格中的「状态」一律不使用彩色，仅通过**灰度层级**区分，保持极简、统一风格。

## 1. 版块一（待确认订单）状态

| 状态   | 含义                     | 背景色     | 边框色     | 文字色   |
|--------|--------------------------|------------|------------|----------|
| 新订单 | 自动生成，未下载效果图/邮件 | `#f5f5f5` | `#e0e0e0` | `#000`   |
| 待确认 | 已下载效果图与邮件       | `#ebebeb`  | `#d0d0d0`  | `#000`   |

- **新订单**：最浅灰，视觉上最「轻」。
- **待确认**：略深一档，表示已进入确认流程。

## 2. 版块二（生产中订单）状态

| 状态   | 含义                     | 背景色     | 边框色     | 文字色   |
|--------|--------------------------|------------|------------|----------|
| 生产中 | 创建订单后默认           | `#d8d8d8` | `#b8b8b8`  | `#000`   |
| 已完成 | 生产并校对完成           | `#333333` | `#333333`  | `#fff`   |

- **生产中**：中灰，比待确认更「重」。
- **已完成**：深灰底 + 白字，最突出，表示该单在生产侧已闭环。

## 3. CSS 变量（便于全局微调）

在 `:root` 中已定义，后续只需改变量即可统一调整：

```css
--status-new: #f5f5f5;
--status-new-border: #e0e0e0;
--status-pending: #ebebeb;
--status-pending-border: #d0d0d0;
--status-producing: #d8d8d8;
--status-producing-border: #b8b8b8;
--status-done: #333333;
--status-done-fg: #ffffff;
```

## 4. 微调建议

- **需要更强对比**：可把 `--status-done` 改为 `#1a1a1a`，或 `--status-new` 改为 `#fafafa`。
- **需要更柔和**：可把 `--status-producing` 改为 `#e0e0e0`，边框改为 `#c8c8c8`。
- **保持无障碍**：已完成（深灰底白字）与背景对比度已满足常规可读性；若需通过 WCAG AA，可把 `--status-done` 调为 `#2d2d2d` 并确保文字为纯白。
