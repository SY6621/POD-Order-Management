# 页面 ↔ 状态 ↔ 按钮 ↔ 快照门禁（定稿 v1.0）

> **关联文档**：`开发文档-订单与生产文档-v0.1.md`（双快照定义、生产文档门禁）。  
> **数据依据**：`orders` 表（Supabase）、`backend/src/models/order.py`、`frontend/src/views/Admin/OrdersShipping.vue`、`frontend/src/stores/orderStore.js`。  
> **说明**：`orders.status` 为**唯一主流程状态**；**不存在** `waiting_shipping` / `pending_logistics` 等单独取值——**「待下物流」= `confirmed`**（见 §1）。

---

## 0. 数据层定稿：`orders.status` 全部取值

| 取值 | 含义 | 典型场景 |
|------|------|----------|
| `new` | 新订单 | 刚抓取，尚未处理 |
| `pending` | 待确认 | 等待客户确认效果图 |
| `effect_sent` | 效果图已发送 | 已发客户，等待确认 |
| `confirmed` | 已确认 | **客户已确认效果图，等待下物流（= 待下物流）** |
| `producing` | 生产中 | 已下 4PX 等，进入生产环节（**见 §1 与 `logistics` 联合判断**） |
| `shipped` | 已发货 | 物流已发出 |
| `delivered` | 已送达 | 客户签收 |
| `cancelled` | 已取消 | 订单取消 |
| `completed` | 已完成 | 订单完成（较少使用） |

**默认值**：`new`。  
**可 NULL**：是（代码未强制 `nullable=False` 时）。

---

## 1. 「待下物流」与「已下物流」如何判定（定稿）

| 业务语义 | 数据层判定 |
|----------|------------|
| **待下物流** | `orders.status = 'confirmed'` |
| **已下物流（进入生产/已创建物流单）** | `orders.status = 'producing'` **且** `logistics.tracking_number` **有值**（以现网逻辑为准） |

**状态流转（简）**：

`pending` / `effect_sent` → `confirmed`（待下物流）→ `producing`（已下物流，需结合运单号）→ … → `shipped` → `delivered` 等。

**代码佐证（物流列表）**：`frontend/src/views/Admin/OrdersShipping.vue`（约 472–476 行）对 `orders` 使用 `.eq('status', 'confirmed')` 查询**待下物流**队列。

**关联表**：`logistics`；`logistics.delivery_status` 取值示例：`pending`（待发货）、`shipped`（已发货）、`delivered`（已送达）——与 `orders.status` 不同维度，**勿混用**。

## 2. 订单主流程（业务语义 ↔ 数据库）— 与快照列

| `orders.status` | 业务称呼（参考） | 设计快照入生产文档 | 物流快照入生产文档 |
|-----------------|------------------|--------------------|--------------------|
| `new` | 新订单 | 否（仅预览） | 否 |
| `pending` | 待确认 | 否，至客户确认 | 否 |
| `effect_sent` | 效果图已发 | 否，至客户确认 | 否 |
| `confirmed` | **待下物流** | 是（已确认设计，可入文档规则以产品为准） | 否 |
| `producing` | 生产中（且通常已有运单） | 是 | 成功下单后 **是** |
| `shipped` / `delivered` / `completed` / `cancelled` | 后段/终态 | 按归档 | 按归档 |

> **已定规则回顾**：设计 SVG **仅客户确认后**进生产文档正式内容；面单 PNG **仅下单成功**进物流区块；失败**不写**物流区块。

## 3. 管理系统：页面 / 路由 ↔ `orders.status` ↔ 按钮

### 3.1 全局

| 路由 | 页面 | 谁可见 | 与订单状态关系 |
|------|------|--------|----------------|
| `/admin/login` | 登录 | 主账号、子账号 | — |
| `/admin/dashboard` | 仪表盘 | 主账号：全店；子账号：授权店 | 汇总各 `status` 数量 |
| `/admin/shops` 等 | 系统管理 | **仅主账号** | — |

### 3.2 待确认订单 `/admin/orders/pending`

| `orders.status` | 列表是否出现 | 主操作 | 设计快照门禁 |
|-----------------|--------------|--------|--------------|
| `new` | ✓ | 出图、进入设计器 | 仅预览 |
| `pending` / `effect_sent` | ✓ | 发客服、客户确认后「创建订单」→ 进入 `confirmed` | 确认前不入正式文档 |
| `confirmed` | 一般**不在**本页主列表 | 应去**物流下单**页 | — |

**按钮与门禁（摘要）**

| 动作 | 结果（数据层） |
|------|----------------|
| 提交效果图 / 发客服 | 常配合进入 `effect_sent` 等（以现网更新逻辑为准） |
| **创建订单**（客户已确认） | → `orders.status = 'confirmed'`（**待下物流**） |

### 3.3 物流下单 `/admin/orders/shipping`

| 数据层 | 列表是否出现 | 主操作 |
|--------|--------------|--------|
| `orders.status = 'confirmed'` | ✓（**待下物流**） | 选渠道、填包裹/申报、**提交 4PX** |
| 下单失败 | 仍为 `confirmed` 或业务另设标记（以现网为准） | 重试；改渠道先取消再下 |
| `producing` 且已有运单 | 一般**不在**本页待办，或归「已处理」 | — |

**按钮与门禁（摘要）**

| 动作 | 快照结果 |
|------|----------|
| 提交下单成功 | 写 `logistics`、面单 PNG → 生产文档物流区块；`status` 进入 `producing`（与现网一致前提下） |
| 失败 | **不写**物流区块 |
