# ETSY订单自动化系统 - 用户沟通偏好规范

## 1. 问题汇总与响应规范

### 1.1 问题记录格式
必须严格使用用户原始问题语句（原话），不得改写、概括或转述。

### 1.2 列表呈现格式
必须以阿拉伯数字编号的纯线性列表形式呈现：
```
1. 第一个问题原话
2. 第二个问题原话
3. 第三个问题原话
```

**禁止使用**：
- 符号列表（`-`、`*`、`•`）
- 段落形式
- 嵌套结构
- 非线性结构

### 1.3 响应明确性
对问题响应须明确为：
- **"立即改进"** - 当前版本立即修复
- **"列入后续迭代"** - 记录到待办清单

**禁用模糊表述**：
- ❌ "会考虑"
- ❌ "后续优化"
- ❌ "可能会"
- ✅ "立即修改"
- ✅ "v2.0版本实现"

## 2. 任务执行透明度规范

### 2.1 三重沟通规范

执行任务时需同步执行：

#### ① 前置准备工作说明
必须明确告知：
- 所需资料
- API服务接入状态
- 账号权限
- 环境配置
- 第三方依赖
- 所有前置条件

#### ② 分步解释机制（适用于关键操作如数据库操作）
- **执行前**：解释原因和预期效果
- **执行后**：确认结果
- **阶段说明**：说明当前在整个开发框架中所处的阶段
- **下一步明确**：明确告知下一步具体操作

#### ③ 手动操作显式提示
当涉及需用户手动执行的操作时：
- 必须清晰标注 **"请手动执行以下命令"**
- 禁止隐式跳过、自动代劳或默认假设用户已理解/完成

**示例**：
```
请手动执行以下命令：

cd D:\ETSY_Order_Automation\backend
poetry run python scripts/test_email.py
```

## 3. PDF交付规范

### 3.1 交付前校对要求
所有PDF生成类任务必须完成完整校对，包括：
- 字体应用
- 文字显示
- 位置精度
- 颜色一致性

### 3.2 交付流程
1. 生成PDF
2. 完整校对
3. 确认无误
4. 交付用户

**严禁**：未校对直接发送

### 3.3 字体斜体不可用时的兜底策略
若指定字体的斜体样式无法正确渲染：
- 必须主动降级使用对应字体的正体（Regular）
- 明确向用户说明该调整
- 禁止因样式不支持而留白、错位或跳过渲染

## 5. 回退与上下文恢复规范

### 5.1 触发条件
当用户声明以下指令时：
- "重新测试前后端对接"
- "重试xxx"
- "回退到xxx阶段"

### 5.2 必须立即执行
1. **同步当前所处阶段**
2. **重载完整任务清单**，包括：
   - SQL执行
   - Vue组件转换
   - 真实数据连接三步
3. **明确重启验证路径与全部前置准备项**：
   - 数据库连接状态
   - API可用性
   - 权限配置完成度

## 7. 初学者友好原则

### 7.1 用户背景
用户为代码开发初学者，处于学习阶段。

### 7.2 教学性要求
- 每个操作都需要详细解释
- 提供命令的作用说明
- 解释技术概念
- 避免假设用户已掌握基础知识

### 7.3 操作指引
- 分步骤说明
- 提供截图参考（如有必要）
- 说明预期结果
- 提供失败时的排查方法
