# ETSY订单自动化系统 - 开发流程规范

## 1. 任务执行透明度规范

### 1.1 任务执行前说明
- 必须说明"为什么这么做"（目的与依据）
- 明确告知所需前置准备（资料、API服务、权限配置等）

### 1.2 任务执行后同步
- 说明当前在整体开发框架中所处的阶段
- 以阶段列表形式清晰呈现进度
- 明确告知下一步具体操作

### 1.3 手动操作显式提示
当涉及需用户手动执行的操作时：
- 清晰标注"请手动执行以下命令"
- 禁止隐式跳过、自动代劳或默认假设用户已理解/完成

项目采用9阶段线性进度表管理开发流程：

```

⏳ 11. 前端界面开发                           ← 当前阶段
```

### 进度报告格式要求
- 使用【阶段名称】作为一级标题
- 每个任务项前加 ✅（已完成）或 ⏳（进行中）
- 当前完成项标注 `← 【当前完成】` 并展开子项验证状态
- 下一步任务标注 `← 下一步`
- 子项使用 `├─`、`└─` 等符号清晰缩进

## 3. 工具链偏好

### 3.1 优先级顺序
1. **终端命令** - 首选方案
2. **Qoder 工具** - 代码编辑、文件操作
3. **VS Code GUI** - 仅在必要时使用，需明确说明理由

### 3.2 命令执行规范
- 提供完整可执行的 PowerShell 命令
- 使用分号 `;` 作为语句分隔符（Windows PowerShell）
- 避免使用 `&&`（PowerShell 不支持）

---

## 4. 代码变更前检查规范

在修改任何代码文件前，必须：
1. 使用 `read_file` 检查文件当前状态
2. 确认是否已有相关代码
3. 避免重复创建或覆盖已有内容

---

## 5. 回退与上下文恢复规范

当用户声明回退/重试指令时，须立即：
1. 同步当前所处阶段
2. 重载完整任务清单
3. 明确重启验证路径与全部前置准备项
   - 数据库连接状态
   - API 可用性
   - 权限配置完成度

---

## 6. 前端开发三阶段流程

每个页面开发遵循：

### 阶段A：静态页面
- 设计页面布局（HTML结构）
- 添加UI组件（Element Plus）
- 使用假数据展示效果

### 阶段B：数据绑定
- 创建 Pinia Store（状态管理）
- 连接 Supabase 获取真实数据
- 替换假数据为真实数据

### 阶段C：功能完善
- 添加交互功能（按钮点击、表单提交）
- 添加数据验证和错误处理
- 优化用户体验（加载状态、提示消息）

## 7. 文件路径规范

### 项目根目录
- 工作区路径：`d:\ETSY_Order_Automation`
- 后端路径：`d:\ETSY_Order_Automation\backend`
- 前端路径：`d:\ETSY_Order_Automation\frontend`

### 资源路径
- SVG模板：`backend\assets\templates`
- 字体文件：`backend\assets\fonts`
- SKU数据：`backend\assets\sku_data`

### 配置文件
- 后端环境变量：`backend\.env`
- 前端环境变量：`frontend\.env`
- Poetry配置：`backend\pyproject.toml`
- npm配置：`frontend\package.json`
