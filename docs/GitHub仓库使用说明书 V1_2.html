四、Rules模块配置

GitHub Automation Rules

自动标记

自动标记Issue和PR

触发工作流

自动化CI/CD流程

代码审查

自动执行代码检查

1 自动标记Bug报告
.github/rules/bug-labeler.yml 
name: 自动标记Bug
description: 当Issue包含特定关键词时自动添加bug标签

on:
  issues:
    types: [opened, edited]

if: |
  issue.body contains 'bug' ||
  issue.body contains '错误' ||
  issue.body contains '崩溃'

then:
  - addLabel: 'bug'
  - addComment: |
      感谢报告Bug！我们会尽快处理。
2 PR自动审查
.github/rules/pr-checks.yml 
name: PR自动检查
description: 对Pull Request进行自动化检查

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  check_description:
    if: pull_request.body == ''
    then:
      - addComment: |
          ⚠️ 请填写PR描述，说明本次修改的内容和目的。
      - addLabel: 'needs-description'
  
  check_large_files:
    if: any(pull_request.files, f.size > 1048576)
    then:
      - addComment: ⚠️ 检测到大于1MB的文件
      - addLabel: 'large-files'
! 代码安全检查
.github/rules/code-quality.yml 
name: 代码质量检查

jobs:
  check_env_files:
    if: |
      any(pull_request.files, f.filename contains '.env' 
          && !f.filename contains '.example')
    then:
      - addComment: |
          🚫 **禁止提交.env文件！**
          这些文件包含敏感信息。
      - addLabel: 'security-issue'
      - requestChanges: true

Deployment Pipeline

5.1 部署架构

1 前端部署到GitHub Pages
.github/workflows/deploy-frontend.yml 
name: 部署前端到GitHub Pages

on:
  push:
    branches: [main]
    paths:
      - 'frontend/**'
      - '.github/workflows/deploy-frontend.yml'
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install & Build
        working-directory: ./frontend
        run: |
          npm ci
          npm run build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./frontend/dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to GitHub Pages
        uses: actions/deploy-pages@v4

六、常见问题与故障排除

FAQ & Troubleshooting

remote: Permission to SY6621/POD-Order-Management.git denied

解决方案：

  1. 检查Token是否过期
  2. 确认Token有 contents:write 权限
  3. 更新remote URL
Terminal 
# 查看冲突文件
git status

# 手动编辑冲突文件，解决冲突后
git add .
git commit -m "解决合并冲突"
git push

可能原因：

  • 仓库未公开
  • vite.config.js中的base路径配置错误
  • 构建产物未正确生成

确保 base: '/POD-Order-Management/' 配置正确

解决方案：

  1. 检查GitHub Secrets是否正确设置
  2. 确认使用的是anon密钥，不是service_role密钥
  3. 检查Supabase的Row Level Security (RLS)策略
