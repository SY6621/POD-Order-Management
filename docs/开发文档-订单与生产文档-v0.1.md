# 开发文档（草案 v0.1）— 订单 / 效果图 / 物流 / 生产文档

> **文档性质**：由产品逻辑与前端交互需求驱动，作为后端与 API 实现的**对齐基线**。  
> **当前版本**：v0.1 — 已固化「双快照」业务规则与格式；字段级接口与枚举待 API 一次性补齐后升级至 v1.0。  
> **修订原则**：前端框架与用户体验优先；后端以本文件中的**状态、快照、区块、门禁**为准实现。

---

## 1. 目标与范围

### 1.1 核心业务三块

| 环节 | 产出物 | 与「生产文档」的关系 |
|------|--------|----------------------|
| 效果图 | SVG（矢量） | 写入生产文档 **设计区块**（见 §3.1 门禁） |
| 物流 | 面单 PNG + 运单号 + 渠道 | 写入生产文档 **物流区块**（见 §3.2 门禁） |
| 生产文档 | 汇总打印/生产依据 | **核心交付物**：聚合 SVG、面单、SKU/刻字、实拍等 |

### 1.2 数据原则（已定）

- **同一套主数据只录入一次**，后续依赖 **规则** 与 **订单快照（Snapshot）** 继承。  
- **前端动线**用于约束后端：列表/详情/按钮对应的**状态迁移**、**只读策略**、**重试**需与 UI 一致。

## 2. 双快照定义（已确认）

### 2.1 设计快照（Design Snapshot）

| 项 | 说明 |
|----|------|
| **格式** | **SVG 矢量图**（用于制作生产雕刻文件） |
| **写入生产文档** | **仅当客户已确认后**，才允许将 SVG **正式写入**生产文档设计区块。 |
| **未确认前** | **仅允许预览**；**不得**作为生产文档正式内容落库。 |

### 2.2 物流快照（Logistics Snapshot）

| 项 | 说明 |
|----|------|
| **格式** | **PNG 位图**；规格 **10×10 cm**；**300 DPI**（用于打印面单）。 |
| **写入生产文档** | **仅当下单成功**后，将面单 PNG 及关联元数据（运单号、渠道等）写入生产文档 **物流区块**。 |
| **下单失败** | **绝不**写入生产文档物流区块（避免脏数据）。 |

### 2.3 实现提示（非约束，供后端/存储）

- 设计侧：除 SVG 文件外，建议存 **版本号、确认时间、确认来源**（便于审计）。  
- 物流侧：除 PNG 外，建议存 **4PX 返回单号、渠道代码、下单时间、操作人**；PNG 存对象存储 URL 或文件 ID。

---

## 3. 生产文档门禁（与快照一致）

### 3.1 设计区块

- **准入**：客户确认 + SVG 定稿版本。  
- **拒绝**：未确认前的 SVG 仅 **预览态**，不进入「可作为生产依据」的文档正式内容。

### 3.2 物流区块

- **准入**：物流下单 **成功**，且已取得符合 §2.2 的面单 PNG。  
- **拒绝**：下单失败、取消未重试成功等情况下，**不写入**物流区块。

### 3.3 整单「可进入生产中作为依据」

- 以产品定义为准：通常需 **生产文档必备区块齐全**（设计 + 物流 + SKU/定制信息等）。  
- 具体 checklist 在 v1.0 与 API 字段对齐时补全。

## 4. 数据血缘（简表）

| 数据阶段 | 说明 |
|----------|------|
| 邮件解析 | 客户名、SKU、刻字、地址等进入 **订单草稿** |
| 待确认 / 设计器 | 生成 SVG；**未确认**不入文档正式内容 |
| 确认 + 创建订单（或等价动作） | 形成 **设计快照** 指针，允许写入文档设计区块 |
| 物流下单 | 成功则形成 **物流快照**（PNG + 元数据），写入文档物流区块 |
| 生产文档 | 引用上述快照，**不重复手抄** |

详细字段级血缘表在 v1.0 与后端字段字典合并后展开。

## 5. 前端驱动后端的约定（草案）

1. **状态机**：页面 `待确认 / 物流下单 / 生产中 …` 与后端 `order_status` **一一映射**，变更由明确操作触发（按钮 + 校验）。  
2. **只读策略**：子账号地址等字段规则已在需求中讨论，落 API 时与本文件 §2、权限矩阵一致。  
3. **幂等与重试**：生产文档写入失败时 **允许重试同一步**；禁止在未取消情况下重复产生有效物流单（与「空单复检」一致）。

**结论（回复产品问题）**  
- **是否清楚**：是 —— 设计快照 = 确认后的 SVG 入文档；未确认仅预览。物流快照 = 成功后的 10×10cm、300DPI PNG 入文档；失败不入。  
- **聊天是否算开发文档**：单独聊天记录**不等同**于正式文档；本文档为**落盘的正式草案**，后续随资料完善升级为 v1.0。  
- **讨论方式**：建议以 **本文件 + 修订** 作为讨论基准；你补充 API 后在同一文档增量更新即可。
